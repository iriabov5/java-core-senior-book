# Инструкция по созданию курса по информационной безопасности

## Начальная настройка

### 1. Создание структуры проекта

```bash
# Создаем основную структуру
mkdir -p info-security-course
cd info-security-course

# Создаем папки для разделов
mkdir -p 01-basics
mkdir -p 02-organizational
mkdir -p 03-legal-regulatory
mkdir -p 04-threat-models
mkdir -p 05-risk-management
mkdir -p 06-incident-management
mkdir -p 07-awareness-training
mkdir -p 08-tools-technologies
mkdir -p 09-audit-compliance
mkdir -p 10-best-practices
mkdir -p 11-real-cases
mkdir -p 12-conclusion

# Создаем основные файлы
touch README.md
touch SUMMARY.md
touch .gitignore
```

### 2. Инициализация Git репозитория

```bash
# Инициализируем git
git init

# Добавляем файлы в первый коммит
git add .
git commit -m "Initial commit: Project structure and basic files"

# Создаем репозиторий на GitHub/GitLab и добавляем remote
git remote add origin https://github.com/your-username/info-security-course.git
git push -u origin main
```

## Процесс работы с Git

### 1. Первый коммит (структура проекта)

```bash
# После создания структуры
git add .
git commit -m "feat: Initial project structure

- Created main directories for course sections
- Added README.md with course overview
- Added SUMMARY.md with table of contents
- Added .gitignore for common files"
git push origin main
```

### 2. Коммиты по разделам

```bash
# После создания каждого раздела
git add 01-basics/
git commit -m "feat: Add basics of information security

- Added overview of information security concepts
- Included CIA triad explanation
- Added threat and vulnerability analysis
- Included security models overview
- Added practical examples and templates"
git push origin main
```

### 3. Коммиты по подразделам

```bash
# Для каждого подраздела
git add 01-basics/threat-models.md
git commit -m "feat: Add threat modeling section

- Added STRIDE methodology explanation
- Included MITRE ATT&CK framework overview
- Added practical threat modeling examples
- Included templates and checklists
- Added tools and resources section"
git push origin main
```

### 4. Обновления и исправления

```bash
# Для обновлений существующих разделов
git add 01-basics/
git commit -m "fix: Update threat modeling section

- Fixed typos in methodology descriptions
- Updated links to latest resources
- Added new examples for cloud security
- Improved templates and checklists"
git push origin main
```

## Шаблоны коммитов

### Для новых разделов:
```
feat: Add [название раздела]

- Added [основные компоненты]
- Included [ключевые элементы]
- Added [практические материалы]
- Included [шаблоны/чек-листы]
```

### Для обновлений:
```
fix: Update [название раздела]

- Fixed [исправления]
- Updated [обновления]
- Added [новые материалы]
- Improved [улучшения]
```

### Для исправлений:
```
fix: Fix [описание проблемы]

- Fixed [конкретное исправление]
- Updated [обновленный контент]
- Corrected [исправленная ошибка]
```

## Советы по работе

### 1. Регулярность
- Делайте коммиты после каждого значимого изменения
- Не накапливайте много изменений в одном коммите
- Пишите понятные сообщения коммитов

### 2. Качество контента
- Проверяйте актуальность нормативных документов
- Включайте практические примеры
- Добавляйте шаблоны и чек-листы
- Ссылайтесь на авторитетные источники

### 3. Структура
- Следуйте единому формату для всех разделов
- Используйте логичную иерархию
- Делайте навигацию удобной
- Добавляйте перекрестные ссылки

### 4. Обновления
- Регулярно обновляйте нормативную базу
- Добавляйте новые угрозы и уязвимости
- Обновляйте инструменты и технологии
- Следите за изменениями в законодательстве

## Полезные команды Git

```bash
# Просмотр статуса
git status

# Просмотр истории коммитов
git log --oneline

# Просмотр изменений в файле
git diff filename.md

# Отмена последнего коммита
git reset --soft HEAD~1

# Создание новой ветки
git checkout -b feature/new-section

# Слияние веток
git merge feature/new-section

# Просмотр веток
git branch -a
```
